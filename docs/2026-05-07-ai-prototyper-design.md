# AI Prototyper — 产品设计文档

> **日期:** 2026-05-07  
> **项目名称:** AI Prototyper（暂定名）  
> **定位:** 面向开发者的AI驱动原型设计工具  
> **状态:** 设计阶段，已通过用户确认

---

## 1. 项目概述

### 1.1 产品定位

AI Prototyper 是一款面向开发者的桌面端原型设计工具，通过结合AI大模型能力，帮助开发者快速将想法转化为可交互的Web原型。

### 1.2 核心价值

- **手绘 → 原型：** 用户可以直接在画板上手绘草图，AI识别后生成规范的UI原型
- **对话式迭代：** 通过自然语言对话逐步调整和完善原型，每次修改实时反映到画板上
- **多页面管理：** 以项目为维度管理多个页面，支持页面间跳转连线
- **可交互导出：** 导出的HTML支持页面跳转等交互，不仅是静态截图

### 1.3 目标用户

开发者（程序员），用于快速生成可交互原型图，偏重代码生成和技术衔接。

### 1.4 平台要求

- Windows桌面客户端，打包为exe安装包
- 基于Tauri 2.x，双击打开独立窗口，不需要浏览器

## 2. 整体架构

应用分为4层，通过Tauri的IPC机制通信：

### 2.1 Tauri壳层（Rust）

- 窗口管理、系统托盘、原生菜单
- 文件系统操作（读写项目文件、导出文件）
- SQLite数据库操作（项目索引、页面元数据、配置存储）
- HTTP请求代理（调用大模型API，避免前端CORS问题）
- 图片处理（HTML转图片、画板导出截图）

### 2.2 前端应用层（React + TypeScript）

- 路由和页面管理
- 状态管理（Zustand）
- 项目/文件管理界面
- 设置和模型配置界面

### 2.3 画板引擎层（Fabric.js）

- 画布核心（缩放、平移、网格、对齐）
- UI组件库（按钮、输入框、卡片、导航栏等预设形状）
- 自由绘制（手绘风格笔触）
- 连线系统（页面间跳转关系的箭头）
- 导出（Canvas转PNG/SVG）

### 2.4 AI集成层

- 模型配置管理（多服务商、多接入方式）
- 对话引擎（聊天式交互界面、上下文管理）
- 记忆系统（用户偏好、可复用模块、项目上下文）
- Skill引擎（预置技能，兼容ClawHub格式）
- 原型生成引擎（将AI返回的描述/HTML解析并渲染到画板）
- 原型修改引擎（解析AI返回的增量修改指令，应用到当前画板状态）

### 3.3 关键数据文件说明

- **canvas.json** — Fabric.js的toJSON()序列化结果，包含所有画板对象的完整状态
- **links.json** — 页面跳转关系：`[{from_page_id, from_element_id, to_page_id, label}]`
- **chat_history.json** — 按页面独立的AI对话历史，保留最近20轮，更早的做摘要压缩
- 每次保存画板时，后台自动生成preview.png缩略图
- API Key/Token在SQLite中加密存储（使用AES加密）
- 项目目录可以直接复制转移，meta.json保证离线可读

### 4.1 编辑器主界面（三栏布局）

### 4.2 左侧面板 — 页面管理（200px宽）

- 页面列表，每项显示名称 + 缩略图
- 显示该页面的跳转连线数量（如"→2"表示有2条跳转）
- 点击切换当前编辑页面
- 顶部"+"按钮新增页面
- 右键菜单：重命名、复制、删除、导出
- 支持拖拽排序（使用dnd-kit）
- 底部显示快照列表（可展开/折叠）

### 4.3 中间区域 — 画板工作区

**画布尺寸预设（工具栏下拉选择）：**
- 桌面端（默认）：1440×900、1280×800、1920×1080
- 平板：1024×768、768×1024
- 手机：375×812、390×844
- 自定义尺寸

默认1440×900桌面画布，以Web端原型为主，兼容移动端设计。

**两种工作模式：**

| 模式 | 说明 |
|------|------|
| 手绘模式（Sketch Mode） | 画笔、自由线条、手写标注为主，适合快速表达想法 |
| 设计模式（Design Mode） | 规范UI组件、精确调整、属性编辑为主，适合精修原型 |

工具栏上有模式切换按钮，两种模式之间随时切换，互不覆盖。

**手绘 → 生成的核心流程：**

1. **手绘阶段** — 用户用画笔工具在画布上自由绘制布局区块、线条、箭头、文字标注（如"导航栏""按钮""三列布局"），也可上传纸拍照片作为底图描摹
2. **AI生成** — 用户点击"AI生成"按钮或在对话框描述需求，AI分析画布上的手绘内容（通过视觉模型截图识别），生成规范的HTML+CSS原型，渲染回画板，自动切换到设计模式
3. **对话式迭代** — 用户在右侧面板对话修改，AI每次修改直接更新画板；用户也可随时切回手绘模式继续补充，再让AI生成

**页面跳转连线：**
- 使用连线工具，从一个页面中的元素拖拽箭头到另一个页面
- 连线在画板上用带箭头的虚线可视化
- 导出HTML时，连线自动转换为可点击的页面跳转链接

### 4.4 右侧面板 — AI对话（300px宽，可折叠）

## 5. AI集成设计

### 5.1 模型分类

| 类型 | 用途 |
|------|------|
| 文字模型（Text Model） | 对话式修改指令、纯文字描述生成原型、增量修改 |
| 视觉模型（Vision Model） | 草图识别、上传图片分析、截图理解 |

配置时分别指定默认文字模型和默认视觉模型。支持两种能力的模型（如GPT-4o、GLM-4V）可同时勾选。

### 5.2 预设服务商

| 服务商 | 文字模型示例 | 视觉模型示例 | 接入方式 |
|--------|-------------|-------------|----------|
| OpenAI | GPT-4o-mini | GPT-4o | 标准API |
| Anthropic | Claude Sonnet | Claude Sonnet | 标准API |
| Google Gemini | Gemini Flash | Gemini Pro | 标准API |
| 通义千问 | Qwen-Max | Qwen-VL-Max | 标准API |
| DeepSeek | DeepSeek-V3 | — | 标准API |
| Moonshot | Moonshot-v1 | — | 标准API |
| 豆包 | Doubao-Pro | Doubao-Vision | 标准API |
| 小米 MIMO | MIMO | MIMO-Vision | 标准API / Token Plan |
| 智谱 GLM | GLM-4 | GLM-4V | 标准API / Coding Plan |

- **标准API：** 用户提供 base_url + api_key + model_name
- **Token Plan（小米）：** 通过小米的token plan机制接入，配置base_url + token
- **Coding Plan（智谱）：** 通过智谱的coding plan方式接入，配置base_url + api_key
- 每个服务商预设好默认base_url，用户只需填认证信息
- 支持"自定义"选项，全部手动填写

### 5.3 配置页面

- 顶部：默认文字模型下拉选择、默认视觉模型下拉选择
- 中部：已配置模型卡片列表，每张卡片显示模型名称、服务商、接入方式、能力类型、默认状态
- 每张卡片有"测试连接"按钮和最近测试状态（✅连接正常 / ❌连接失败）
- 底部："+ 添加新模型"按钮

**添加/编辑模型流程：**
1. 选择服务商（下拉列表，含"自定义"）
2. 选择接入方式（根据服务商动态显示：标准API / Token Plan / Coding Plan）
3. 填写认证信息（根据接入方式动态显示字段）
4. 勾选模型能力：文字 / 视觉 / 两者
5. 点击"测试连接"

**测试连接功能：**
- 文字模型：发送测试Prompt（如"回复OK"），验证能否正常返回
- 视觉模型：发送预设测试小图片 + "描述这张图片"，验证多模态能力
- 显示测试结果：✅ 连接正常（响应XXms）/ ❌ 连接失败（错误信息）
- 测试结果缓存显示在配置列表上

### 5.4 AI调用的两套Prompt策略

**任务1：草图识别 → 原型生成**
- 输入：画布手绘内容截图 + OCR提取的手写文字标注
- 处理：AI通过视觉模型分析草图的结构和意图
- 输出：结构化的HTML+CSS代码 + 组件描述JSON
- 前端接收后渲染到画板，转为Fabric.js对象

**任务2：对话式修改**
- 输入：当前画板状态截图（或序列化JSON）+ 用户修改指令 + 对话历史
- 处理：AI理解当前原型状态和用户意图
- 输出：增量修改指令（修改哪些元素、怎么改）或完整替换HTML
- 前端解析指令，应用到画板对应元素

### 5.5 对话上下文管理

每次AI调用携带的上下文（按优先级排序）：
1. 系统Prompt（角色定义、输出格式要求）
2. 项目设计规范（从记忆系统加载）
3. 用户偏好（从记忆系统加载）
4. 当前画板状态（截图 + 关键元素的JSON描述）
5. 页面元信息（尺寸、类型：桌面/移动）
6. 对话历史（最近20轮，更早的做摘要压缩）
7. 用户当前输入（文字 / 上传的草图图片）

上下文通过关键词和标签匹配注入，只加载相关部分，避免token浪费。

### 5.6 AI输出规范

AI返回结构化JSON，前端统一解析：

```json
{
  "type": "generate | modify",
  "html": "<div class='prototype'>...</div>",
  "css": ".prototype { ... }",
  "interactions": [
    {"element": "#btn-detail", "action": "navigate", "target": "page-uuid"}
  ],
  "memory_updates": {
    "preferences": {"style": "简约", "layout": "侧边导航"},
    "design_system": {"primary_color": "#7c6aef"}
  },
  "skill_used": "landing-page",
  "message": "已生成首页原型，包含导航栏、轮播图和产品列表。"
}
```

| 字段 | 说明 |
|------|------|
| html/css | 原型完整代码，在画板中渲染为真实DOM |
| interactions | 可交互元素定义，用于导出HTML时生成点击跳转 |
| memory_updates | 记忆更新指令，由记忆系统自动处理 |
| skill_used | 当前使用的skill标识（如果有） |
| message | AI回复文案，显示在对话框中 |

## 6. AI记忆系统

### 6.1 三层记忆架构

**第一层：用户偏好记忆（全局，跨项目）**

| 偏好维度 | 示例 |
|---------|------|
| 设计风格 | 简约/复杂、扁平/拟物 |
| 配色偏好 | 冷色/暖色/中性、主色倾向 |
| 布局偏好 | 顶部导航/侧边导航、栅格列数、间距紧凑/宽松 |
| 组件偏好 | 按钮圆角/直角、卡片样式、表单横向/纵向 |
| 字体偏好 | 标题字重、正文字号、行间距 |

积累方式：
- 自动积累：通过用户对话中的反馈提取（"我不喜欢圆角按钮""导航栏放左边"）
- 手动配置：设置页面中提供偏好配置入口

**第二层：可复用模块记忆**

- 用户设计满意的功能模块后，标记为"收藏模块"
- AI自动提取模块特征（布局结构、组件组合、配色方案），生成模块摘要
- 下次用户描述类似需求时，AI优先参考已收藏模块
- 模块库在设置中可浏览、搜索、删除
- 示例：收藏了一个满意的后台数据统计卡片，下次说"加一个数据统计区"时AI直接复用

**第三层：项目上下文记忆**

| 记忆类型 | 说明 |
|---------|------|
| 项目设计规范 | 已确定的配色方案、字体、间距规范，后续生成保持一致性 |
| 页面关联记忆 | 已有哪些页面，生成新页面时保持风格统一 |
| 迭代历史 | 每个页面的设计演变过程，避免AI反复提出已否决的方案 |
| 项目术语 | 用户提到的产品相关术语及对应含义 |

### 6.2 记忆存储结构

参见3.2节目录结构中的 `memory/` 目录。

### 6.3 记忆调用机制

- 用户发送对话指令时，自动注入相关记忆到AI上下文
- 注入优先级：项目设计规范 > 用户偏好 > 可复用模块 > 迭代历史
- 记忆通过关键词和标签匹配，只注入相关部分，避免token浪费
- AI输出中包含memory_updates字段，记忆系统自动更新

## 7. AI Skill系统

### 7.1 定位

- 预置的专业原型设计技能，每个skill是一套结构化的Prompt + 输出规范
- 用户在对话中通过关键词自动触发，或手动 `@skill名称` 调用
- 不可新增、不可编辑，由应用版本更新迭代
- 兼容ClawHub格式，后续可对接ClawHub的skill生态

### 7.2 预置Skill清单

| Skill名称 | 触发关键词 | 功能说明 |
|-----------|-----------|---------|
| **landing-page** | 落地页、首页、Landing | 生成完整落地页原型：Hero区、特性展示、CTA、Footer，根据行业自动适配内容 |

## 8. 原型导出与项目功能

### 8.1 单个页面导出

| 格式 | 说明 |
|------|------|
| HTML | 独立HTML文件，内联CSS无外部依赖，包含页面跳转交互，自适应viewport |
| PNG | 页面截图，可选1x/2x/3x分辨率 |

HTML导出可选：生成preview文件夹，包含HTML和引用的资源图片。

### 8.2 批量导出（项目级别）

**HTML批量导出：**

```
export_项目名_20260507/
├── index.html           ← 入口页，包含页面导航目录
├── page-首页.html
├── page-商品详情.html
├── page-购物车.html
└── assets/              ← 共享资源（图片等）
```

- index.html为项目导航页，列出所有页面并可点击跳转
- 每个页面HTML中的跳转连线自动转换为页面间链接

**PNG批量导出：** 每个页面生成一张PNG，放在同一文件夹中。

### 8.3 导出配置项

- 导出格式：HTML / PNG / 两者都导出
- 图片分辨率：1x / 2x / 3x
- HTML选项：是否包含交互跳转、是否生成导航首页
- 导出路径选择（默认桌面，可自定义）

### 8.4 项目管理

| 功能 | 说明 |
|------|------|
| 新建项目 | 输入名称、描述、选择默认画布尺寸 |
| 打开项目 | 最近项目列表 + 浏览所有项目 |
| 项目概览 | 卡片网格展示所有页面缩略图，拖拽重排，快速预览，跳转关系概览图，批量选择导出 |
| 项目复制 | 复制整个项目作为新项目 |
| 项目导包 | 导入他人项目文件夹（读取meta.json + canvas.json重建） |

### 8.5 自动保存

- 每次画布变更后自动保存canvas.json
- 每次对话新增消息后自动保存chat_history
- 退出页面时自动生成preview.png缩略图
- 保存状态显示在底部状态栏

### 8.6 设计快照

- 用户可标记当前画板状态为"快照"
- 快照列表在页面管理面板底部显示（可展开/折叠）
- 后续修改不满意可回退到任意快照

## 附录：关键交互流程

### A.1 新用户首次使用流程

1. 打开应用 → 欢迎引导页
2. 进入设置 → 配置AI模型（至少配置一个文字模型或视觉模型）
3. 测试连接确认可用
4. 新建项目 → 输入名称，选择画布尺寸
5. 进入编辑器 → 选择手绘模式或直接在对话框描述需求
6. 开始设计

### A.2 手绘草图生成原型流程

1. 切换到手绘模式
2. 用画笔/形状工具绘制布局草图，写文字标注
3. （可选）上传纸拍照片作为底图描摹
4. 点击"AI生成"或在对话框说"根据我的草图生成原型"
5. AI通过视觉模型识别手绘内容
6. 生成规范HTML+CSS原型渲染到画板
7. 自动切换到设计模式
8. 用户通过对话继续迭代修改

### A.3 对话式修改流程

1. 在右侧对话框输入修改指令（文字描述）
2. AI结合当前画板状态 + 记忆系统上下文理解意图
3. （自动触发skill时，按skill定义的步骤执行）
4. AI返回增量修改指令或完整HTML
5. 前端解析并应用到画板
6. AI回复标注"已更新 ✅"
7. 自动保存画板状态和对话历史
8. 记忆系统自动更新偏好/项目上下文
